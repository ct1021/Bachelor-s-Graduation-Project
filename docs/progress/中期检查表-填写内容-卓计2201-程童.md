## 一、毕业设计（论文）工作进度

### （一）文献调研与开题阶段（2025年11月 — 2026年1月）【已完成 ✅】

1. **文献调研与方案设计**：深入研读了中外文核心文献，涵盖ZMP判据、MPC控制、深度强化学习PPO、模仿学习DeepMimic、Sim2Real域随机化等关键方向。明确了以"运动学重定向技术处理步态数据，以PPO算法进行策略微调"的核心技术方案。
2. **开题答辩与归档**：于2026年1月9日顺利通过开题答辩，确立了"模仿学习初始化+强化学习微调"的核心技术路线。
3. **仿真环境与强化学习跑通**：完成了PyBullet与MuJoCo双仿真引擎支持的搭建。初期使用 `CartPole` 倒立摆作为双足平衡的简化基线模型，成功跑通 PPO 算法并输出 TensorBoard 收敛曲线，验证了深度强化学习框架的可行性。

### （二）算法建模与前期仿真验证阶段（2026年1月 — 2026年2月）【已完成 ✅】

1. **双足模型早期物理适配**：
   - 参照宇树 G1 的真实比例，自主搭建了 **6-DoF 简化双足模型**（躯干 + 左右腿各 3 关节）。
   - 开发完整的 URDF 验证与静态重力测试工具链（`scripts/validate_urdf.py`），通过了 PyBullet 的关节限位校准与质量分布测试，为引入真实全尺寸机器人打下模型基础。
2. **离线步态先验网络（落实导师指导要求）**：
   - 响应导师关于"离线小模型+KL散度"的指导建议，设计并实现了基于 PyTorch 的轻量级 MLP 策略网络（参数量约 12K），作为步态先验模型。
   - 开发了混合损失函数（`scripts/hybrid_loss.py`），实现了 `Loss = BC_Loss + λ × KL_Divergence` 的多目标优化闭环，确保在线策略不偏离先验分布。
3. **多目标物理感知奖励函数构建**：
   - 独立设计了 `PhysicsRewardWrapper` 包装器。包含速度跟踪、能耗惩罚(COT) 和 CoP稳定性约束三大项奖励分支，为双足姿态提供强物理学约束。

### （三）全尺寸人形机器人(G1)接入与实机部署准备阶段（2026年3月 — 至今）【进行中 🔄】

1. **高精度 MuJoCo G1 仿真环境开发**：
   - 成功对接 Unitree G1 官方开源架构，弃用早期简化模型，全面转入 23-DoF G1 机器人的高精度 MJCF 进行强化学习环境封装。
   - 开发 G1 双足行走自定义 Gymnasium 环境（`envs/g1_env.py`）。完善了多维观测空间（关节位姿速率+IMU四元数）及输出截断的动作力矩空间。
2. **Sim2Sim 跨引擎鲁棒性自证（域随机化）**：
   - 实现 Sim2Sim 域随机化校验体系（`scripts/sim2sim_dr_eval.py`）。
   - 在关节摩擦力与质量属性抖动 20% 的噪声条件下，评估了 PPO Agent 基线模型的动作收敛度与速度保持能力，并成功绘制了物理干扰图表（`Midterm_Evidence/3_Sim2Sim_DR_Evaluation`）。
3. **视觉评估与离屏渲染**：
   - 开发了基于 EGL 无头模式的 `render_g1_demo.py` 脚本，可在无显示器的 Linux 服务器通过 GPU 渲染高帧率 MP4 视频评估样本，充实中期答辩素材。
4. **Linux GPU 工作站与实机底层网络打通（极其重要）**：

## 二、存在的问题及拟采取的措施

### 存在的主要问题：

1. **G1 灵巧手拆卸与实机调试延后**：G1灵巧手的拆卸需要专业工具和时间，实机部署计划延后至仿真验证完成后再启动。

2. **MoCap步态数据有待获取**：需与电子科技大学课题组师兄确认数据格式与采集方案，当前先用正弦函数模拟参考步态推进算法开发，后续引入CMU MoCap公开数据集进行pipeline验证。

3. **Sim2Sim跨引擎验证尚未开展**：MuJoCo训练的策略需在PyBullet等不同物理引擎中进行交叉验证，证明策略的泛化性（任务书中"Sim环境验证"要求）。

### 拟采取的措施：

1. **仿真优先策略**：集中利用Linux GPU工作站开展MuJoCo Headless训练，3月15日前产出Reward曲线和步态渲染视频，确保中期答辩有充分的可视化成果支撑。

2. **分阶段数据引入**：先用正弦参考步态验证训练pipeline → 引入MoCap舞蹈数据测试运动学重定向（Retargeting） → 最终使用人体行走步态数据进行正式训练。选用人体步态数据的原因：（1）人体步态经进化优化，能量效率最高；（2）天然满足ZMP/CoP稳定性约束；（3）为RL提供安全的探索边界，避免灾难性动作。

3. **Sim2Sim验证路线**：完成MuJoCo训练后，将策略模型导出并在PyBullet中加载验证，对比跨引擎的速度跟踪误差与步态对称性指标，证明策略在不同仿真环境下的可迁移性。

4. **实机部署准备**：网线与交换机已就位，待灵巧手拆卸完成后，通过CycloneDDS + unitree_sdk2进行实机通信测试，开展Sim2Real部署。

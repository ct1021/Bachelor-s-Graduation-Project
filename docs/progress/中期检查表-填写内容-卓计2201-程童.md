## 一、毕业设计（论文）工作进度

### （一）前期探索与算法基础阶段（2025年11月 — 2026年1月）【已完成 ✅】

刚开始做这个课题的时候，我花了不少时间把 ZMP、模型预测控制(MPC) 还有强化学习 PPO 的老方法捋了一遍。最后跟导师碰完，决定用“模仿学习打底，然后再用强化学习去微调”的思路。1月份把开题答辩给过了。

因为刚接手，没敢直接碰全尺寸机器人的环境。我就自己写了个简易版的 `CartPole`（倒立摆），拿来测试 PPO 算法能不能把闭环控制跑通。当时环境连到了 TensorBoard 上，看到奖励分数往上跑，基本上确定底层的网络框架没什么毛病了。
> 📌 **【这里插入图片：早期算法基线测试】**
> 说明：把刚跑出来的 CartPole TensorBoard 截图放这儿。
> 获取方式：在 Windows 电脑开个终端跑 `tensorboard --logdir=logs/PPO_1 --port=6006`，浏览器开 `localhost:6006` 截个图就行，主要展示那个 Reward 上升曲线。

1月份期间还顺便把通用价值函数逼近器(UVFA) 和混合专家模型(MoE) 的论文看了一下，弄懂了怎么在神经网络里做状态拆分，这基本是在给后面写 G1 自己的网络铺路。

### （二）双足模型自建与离线网络开发（2026年1月 — 2026年2月）【已完成 ✅】

寒假没法去深圳实验室调实机。我干脆在自己电脑上拿 PyBullet 写了个 6 个自由度的极简双足 URDF 模型（就是个身子下面挂两根三段式的腿）。拿它测了重力、关节限位，算是跑通了物理仿真的流程。

这期间的大工程是**按照张老师的要求写了个离线步态网络**。
我自己写了个很小的 MLP 策略网络（参数才 1万出头），也就是导师说的“离线小模型”。为了防止在线强化学习策略训飞了，我在计算总损失的时候，强行拿 BC 模仿损失加上了一个 KL 散度（也就是混合损失函数），逼着机器人的实时动作不敢偏离离线经验太多。同时还加了各种速度和能耗（COT）惩罚。这些代码跑通的时候确实不容易，光调权重参数就调了好久。

### （三）G1 实机接入与大模型仿真（2026年3月 — 至今）【进行中 🔄】

这大半个月属于干活密度最高的一段时间，因为终于要去接实验室硬件了：

1. **废弃极简模型，转向官方开源架构**：我直接把原来 6 自由度的东西全扔了，用上了宇树刚放出来的 23 自由度原厂高精度 MJCF 模型，在 MuJoCo 里面重新写了动作空间和观察空间的接口逻辑。
2. **自己做抗干扰测试**：为了证明算法有用，我写了个 Python 脚本去故意给仿真器里的各个关节加高斯噪声（摩擦力和重量上下抖动 20%）。结果 PPO 模型抗住了，动作还算连贯。
> 📌 **【这里插入图片：Sim2Sim 抗干扰鲁棒性图表】**
> 说明：把 `Midterm_Evidence/3_Sim2Sim_DR_Evaluation` 文件夹里那两张抖动的曲线图插进来一张。
3. **把无头渲染搞通了**：由于服务器上跑代码连个显示器都没有，我写了个用 EGL 偷偷在后台靠 GPU 把机器人走路画面录成 MP4 的脚本。中期答辩的视频素材算是搞定了。

**4. 硬件底层网络打通（这步踩的坑最多）**：
想在 Linux 上直连真的 G1 机器人，各种配置问题接踵而至。
   - 保险起见，我们先把 G1 上非常容易摔坏的灵巧手给拆了。
   - 彻底配好了 Ubuntu 22.04 高性能主机的开发环境，装了一大堆依赖库。
   - **坑点与警告记录**：在 Windows 和 Linux 之间导代码，网路特别折磨人。在 Linux 用 Git 的时候必须开代理 `export http_proxy=http://你的Windows局域网IP:7890`，不然在 GitHub 上啥都拉不下来。由于双机开发容易产生代码冲突，有时候还得用 `git reset --hard` 去强拉远端覆盖本地。更要命的是 `conda activate biped_rl` 永远别忘了敲，不然满屏幕包找不到的错！
   - 目前，我用网线把这台 Linux 电脑跟 G1 主控给绑死了（配了 `192.168.123` 的静态 IP 段），用官方最新的 CycloneDDS SDK 调通了数据接口（`LocoClient`）。我自己写的心跳抓取脚本已经能毫秒级地读机身状态了。

## 一、毕业设计（论文）工作进度

### （一）文献调研与开题阶段（2025年11月 — 2026年1月）【已完成 ✅】

1. **文献调研与方案设计**：深入研读了15篇以上中外文核心文献，涵盖ZMP判据（Vukobratović, Kajita）、MPC控制（Di Carlo）、深度强化学习PPO/SAC（Schulman）、模仿学习DeepMimic（Peng）、Sim2Real域随机化（Hwangbo, Tobin）以及安全约束CBF（Xu）等关键方向，系统梳理了IL与RL融合技术路线，确定了以运动学重定向技术处理步态数据、以PPO算法进行策略微调的核心技术方案。
2. **开题答辩与归档**：于2026年1月9日顺利通过开题答辩，根据专家组意见修订完善了《开题报告》，确立了"模仿学习初始化+强化学习微调"的核心技术路线。
3. **仿真环境基础搭建**：完成了PyBullet与MuJoCo双仿真引擎支持的配置，安装并验证了Gymnasium强化学习接口。

### （二）算法建模与仿真验证阶段（2026年1月 — 至今）【进行中 🔄，已完成约75%】

1. **毕设工程化体系建设**：
   - 在GitHub初始化了标准化项目仓库（Bachelor-s-Graduation-Project），严格按科研规范划分：`data/`（步态数据）、`envs/`（URDF/MJCF模型）、`scripts/`（算法脚本）、`models/`（模型权重）、`logs/`（TensorBoard日志）、`docs/`（文档与图表）。
   - 配置了完整的`requirements.txt`环境清单与版本控制规则，确保实验可复现。

2. **强化学习基线训练与可视化**：
   - 完成了基于PPO算法的倒立摆（CartPole-v1）平衡控制基线训练；集成TensorBoard实现训练过程量化追踪，训练曲线显示平均奖励稳步攀升，验证了PPO算法闭环的收敛性。

3. **离线步态先验模型开发（落实导师建议）**：
   - 根据张晓蓉导师关于"离线小模型+KL散度损失"的指导，实现了轻量级步态先验网络GaitPriorNetwork（MLP架构，~4K参数）、混合损失函数HybridLoss（Loss = BC_Loss + λ×KL_Divergence）以及物理感知奖励函数PhysicsRewardWrapper（速度跟踪+COT能耗+CoP稳定性），全部代码通过集成测试。

4. **成功对接G1开源RL生态**：
   - 调研并对接了Unitree G1官方开源仓库（unitree_mujoco、unitree_rl_gym/mjlab），获取了G1人形机器人的高精度MuJoCo MJCF模型文件。
   - 基于开源社区资源，参考成熟的G1强化学习训练方案（如MuJoCo Playground、MimicKit参考运动框架），为本课题的仿真训练提供了可靠的数据与工具支撑。

5. **基于MuJoCo的高精度仿真环境搭建**：
   - 开发了G1双足行走自定义Gymnasium环境（G1WalkEnv），实现了完整的观测空间（关节位姿+速度+躯干姿态）、动作空间（归一化关节力矩）及多目标奖励函数。
   - 奖励函数体现"模仿学习+强化学习"融合特色：核心项为模仿学习跟踪奖励（Tracking Reward，计算当前动作与参考轨迹的MSE距离），辅以速度跟踪、存活奖励、能耗惩罚（COT）及姿态保持约束。
   - 内置正弦参考步态生成器（模拟双足行走的周期性运动），后续将替换为CMU MoCap人体步态数据。

6. **实现基于离屏渲染的视觉评估手段**：
   - 开发了MuJoCo离屏渲染脚本（render_g1_demo.py），支持EGL/OSMesa无头模式自动检测，能够在无图形桌面的Linux服务器上通过GPU加速渲染，导出G1步态仿真MP4视频——为中期答辩提供直观的可视化素材。

7. **PPO + 模仿学习训练框架搭建**：
   - 基于Stable-Baselines3构建了完整的PPO训练流水线（train_g1_ppo.py），集成了TensorBoard日志、定期Checkpoint保存、独立评估环境等工业级训练功能。
   - 策略网络采用双层256节点MLP架构，输入融合关节本体感知，输出关节力矩控制信号。

8. **Linux高性能工作站搭建与SDK部署**：
   - 搭建了基于Ubuntu 22.04的GPU计算节点（RTX 2070 Ti），通过Tailscale组网实现远程开发。
   - 在biped_rl环境中完成PyTorch+CUDA、MuJoCo、Stable-Baselines3等全套深度学习工具链部署。
   - 编译安装CycloneDDS + unitree_sdk2_python，预留Sim2Real实机通信接口。
   - **网线、交换机等硬件已就位**，具备实体联调的物理条件。

## 二、存在的问题及拟采取的措施

### 存在的主要问题：

1. **G1 灵巧手拆卸与实机调试延后**：G1灵巧手的拆卸需要专业工具和时间，实机部署计划延后至仿真验证完成后再启动。

2. **MoCap步态数据有待获取**：需与电子科技大学课题组师兄确认数据格式与采集方案，当前先用正弦函数模拟参考步态推进算法开发，后续引入CMU MoCap公开数据集进行pipeline验证。

3. **Sim2Sim跨引擎验证尚未开展**：MuJoCo训练的策略需在PyBullet等不同物理引擎中进行交叉验证，证明策略的泛化性（任务书中"Sim环境验证"要求）。

### 拟采取的措施：

1. **仿真优先策略**：集中利用Linux GPU工作站开展MuJoCo Headless训练，3月15日前产出Reward曲线和步态渲染视频，确保中期答辩有充分的可视化成果支撑。

2. **分阶段数据引入**：先用正弦参考步态验证训练pipeline → 引入MoCap舞蹈数据测试运动学重定向（Retargeting） → 最终使用人体行走步态数据进行正式训练。选用人体步态数据的原因：（1）人体步态经进化优化，能量效率最高；（2）天然满足ZMP/CoP稳定性约束；（3）为RL提供安全的探索边界，避免灾难性动作。

3. **Sim2Sim验证路线**：完成MuJoCo训练后，将策略模型导出并在PyBullet中加载验证，对比跨引擎的速度跟踪误差与步态对称性指标，证明策略在不同仿真环境下的可迁移性。

4. **实机部署准备**：网线与交换机已就位，待灵巧手拆卸完成后，通过CycloneDDS + unitree_sdk2进行实机通信测试，开展Sim2Real部署。

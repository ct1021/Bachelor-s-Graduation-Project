# Phase 3: Sim2Real 与硬件实机部署日程安排 (3月12日 - 3月15日)

考虑到目前强化学习在 MuJoCo 仿真跑通进度远超预期（仅耗时至周四），我们完全可以利用本周剩余时间**超前推进 Sim2Sim 跨引擎验证和物理真机的接管准备**。这不仅能极大地扩充中期答辩的工作量，也为下半学期的真机测试扫清硬件障碍。

以下是为您定制的冲刺期（冲刺中期答辩）具体行事历：

## 📅 周四 (今日) - 仿真数据收尾与 SDK 接入预研
**目标：完成软资产积攒，开始看透实机通信底层。**

1. **查收 PPO 模型**：等待服务器端 50万步的 `train_g1_ppo.py` 跑完，回收最终生成的 `models/g1_ppo_50k.zip` 和 Reward 曲线截图。
2. **生成最终仿真验证视频**：使用刚刚产出的 `g1_ppo_50k.zip` 跑通测试，输出带有 RL 补偿的步行视频。
3. **Unitree SDK 2 联调预研**：查阅官方 SDK 文档，明确 CycloneDDS 组网格式。

## 📅 周五 (明日) - Sim2Sim 跨引擎鲁棒性验证 (补充极棒的学术工作量) - **[已完成]**
**目标：证明我们的算法不仅仅“过拟合”在 MuJoCo 上，具备真实世界的泛化潜力。**

1. **[✅] PyBullet 基线验证**：将训练好的 `models/g1_ppo_50k.zip` 导入到之前写的 PyBullet 环境（或者 Isaac Gym）中。
2. **[✅] 域随机化测试**：在仿真环境中对机器人的各个关节主动加入 `20%` 的物理噪声（摩擦力、质量抖动），观察策略的稳定性。
3. **[✅] 撰写 Sim2Sim 章节报告**：截图并把这部分高级的鲁棒性分析直接塞进中期报告里，这在本科毕设中是非常亮眼的加分项。

## 📅 周六 - 硬件拆卸与实机物理准备 (关键节点)
**目标：排除物理损坏隐患，正式接管 G1。**

1. **[✅] 上午：拆卸 G1 灵巧手 (Dexterous Hands)** ⚠️**核心操作**
   - **原因**：由于早期 RL 步态极易发生严重摔倒侧翻。G1 的三指/五指灵巧手极其精密，撑地易导致机械结构断裂。
   - **实操更新 (3/13)**：两只灵巧手已成功拆除。但遇到以下突发情况：
     - 原厂组装有上胶，拆除时导致一侧原装螺丝断裂（滑丝并卡在螺纹处）。
     - 缺失普通手对应的硬件模块，同时考虑螺丝现状与“电火花”强行取断丝的风险，**导师与团队决定放弃更换假手，直接以“无手”状态进行双足实验**。
     - *影响评估*：本项目核心是双足强化学习步态(Locomotion)，手臂主要做配重和动量平衡(0扭矩下垂)，不带手不影响核心实验结论，且大幅降低了侧翻经济风险。此方案完全可接受。
2. **下午：有线网络拓扑打通**
   - 用准备好的网线和交换机，将 Linux 工作站与 G1 背部的调试网口进行物理直连。
   - 配置工作站静态 IP（例：`192.168.123.X`），使用 `ping` 命令验证是否能稳定 PING 通机器人内置的主控（Raspberry Pi / Jetson）。

## 📅 周日 - 答辩前夜整理
**目标：整合所有证据链，无懈可击地迎接答辩。**

1. 导出所有的曲线图、仿真录屏、网线连接真机的照片、灵巧手拆卸收纳的照片。
2. 串联进 PPT 中。明确展示我们的路线是：基于开源数据的**离线模型冷启动** -> **MuJoCo 大规模训练** -> **Sim2Sim 鲁棒验证** -> **真机网络连通并做好摔机防护准备**。
